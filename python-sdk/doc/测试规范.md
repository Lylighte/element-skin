# 测试规范

SDK 自身使用 `httpx.MockTransport` 和精确请求断言。第三方应用测试也可以使用同样方式。

## Mock HTTP 请求

```python
import httpx

from element_skin_sdk import ElementSkinAPI

def handler(request: httpx.Request) -> httpx.Response:
    assert request.method == "GET"
    assert request.url.path == "/v1/users/me"
    assert request.headers["authorization"] == "Bearer access-token"
    return httpx.Response(200, json={"id": "user-1"})

transport = httpx.MockTransport(handler)

api = ElementSkinAPI(
    "https://skin.example.com",
    access_token="access-token",
    transport=transport,
)

assert api.me() == {"id": "user-1"}
```

## SDK 测试命令

```bash
python -m pytest
```

## 覆盖率门槛

SDK 要求 100% 行覆盖率和 100% 分支覆盖率：

```bash
python -m coverage run -m pytest
python -m coverage report -m
```

覆盖率配置在 `pyproject.toml` 中：

```toml
[tool.coverage.run]
branch = true
source = ["element_skin_sdk"]

[tool.coverage.report]
fail_under = 100
show_missing = true
```

## 断言要求

测试应断言精确值：

- HTTP method
- 路径
- 查询参数
- JSON 或 form body
- Authorization header
- 错误类型
- 错误状态码
- 错误响应体
- token 存储副作用

除非“非空”本身就是业务定义，否则不要使用“返回不为空”这类弱断言。

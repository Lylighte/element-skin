# 权限模型

Element Skin 将站点能力表示为权限代码。SDK 提供明确的权限常量，避免第三方
Python 应用到处手写字符串。

## 导入权限常量

```python
from element_skin_sdk.permissions import (
    AccountScopes,
    MinecraftScopes,
    ProfileScopes,
    TextureScopes,
    WardrobeScopes,
)
```

常见权限：

```python
AccountScopes.READ_SELF
ProfileScopes.READ_OWNED
TextureScopes.READ_PUBLIC
WardrobeScopes.ENTRY_APPLY_OWNED
MinecraftScopes.SESSION_HASJOINED_SERVER
```

## 用户委托权限

Authorization Code + PKCE 和 Device Code Flow 都代表用户委托。

允许请求的范围包括用户自己的资源、公开读取能力，以及该用户本身具有的委托式管理
能力。

SDK 会拒绝用户委托流程请求 `server` 和 `system` 范围权限。

```python
oauth.authorization_url([MinecraftScopes.SESSION_HASJOINED_SERVER])
# 抛出 InvalidScope
```

## 受保护主体

`protected` 是用户主体属性，不是 OAuth scope，也不是角色。`/v1/users/me` 和管理员用户接口会返回该字段。

前端可以把 `protected = True` 的管理员展示为“超级管理员”。第三方应用不能申请 `protected`，也不能通过角色获得该状态；受保护管理权限必须由站点管理员在服务端转让。

## Client Credentials 权限

Client Credentials 代表应用以自己的权限主体访问。

SDK 允许该流程请求 `public`、`server` 和经管理员审核授予应用主体的 `any`
范围权限。`any` 权限用于应用自身的后台能力，例如管理员授权的外部工具管理邀请码。

```python
oauth.client_credentials([MinecraftScopes.SESSION_HASJOINED_SERVER])
```

```python
from element_skin_sdk.permissions import InviteScopes

oauth.client_credentials([InviteScopes.READ_ANY])
```

SDK 会拒绝依赖用户上下文的委托权限：

```python
oauth.client_credentials([AccountScopes.READ_SELF])
# 抛出 InvalidScope
```

Client Credentials 不读取用户角色，也不能通过用户授权获得权限。服务端最终仍会检查
应用状态、应用申请权限、`client:{client_id}` 权限主体和本次 token 请求 scope。

## OAuth 生命周期

用户委托授权和应用状态会受到站点权限变化影响：

- 授权用户权限或角色变化后，如果已有授权不再满足权限条件，服务端会撤销对应 grant。
- 应用创建者权限或角色变化后，如果应用申请的非 `server` 权限不再满足创建条件，服务端会停用该应用。
- 仍然满足条件的授权和应用不会被误伤。
- 用户注销或管理员删除用户时，服务端会清理该用户授予的 OAuth 授权、设备码、提交的应用和应用权限主体。

因此，第三方应用必须处理 `401`、`403`、`invalid_grant`、`invalid_client` 等失效场景，
不要假设授权或应用审核状态永久有效。

## 本地权限校验

`TokenSet.permissions` 保存服务端返回的权限列表。

当 `ElementSkinAPI` 获取到已知权限列表时，会在请求前检查所需权限。

```python
from element_skin_sdk import ElementSkinAPI
from element_skin_sdk.models import TokenSet

tokens = TokenSet(
    access_token="token",
    token_type="Bearer",
    expires_in=3600,
    permissions=("profile.read.owned",),
)

api = ElementSkinAPI("https://skin.example.com", token=tokens)
api.me()
# 抛出 PermissionDenied，因为缺少 account.read.self
```

如果权限列表未知，SDK 会发送请求，由服务端做最终权限判断。

## 权限目录

SDK 可以加载服务端返回的权限元数据：

```python
from element_skin_sdk.permissions import PermissionCatalog

catalog = PermissionCatalog.from_api_payload(
    {
        "permissions": [
            {
                "code": "account.read.self",
                "category": "account",
                "action": "read",
                "scope": "self",
            }
        ]
    }
)
```

将 `PermissionCatalog` 传给 `PermissionValidator` 后，SDK 可以在本地拒绝未知权限。
